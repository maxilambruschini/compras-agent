<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| inbound message (untrusted) → orchestrator | Free-form text + message_id drive state transitions and the write gate |
| concurrent deliveries → conversation row | Two messages for the same sender could race on state |
| draft_gasto JSON → DraftGasto.model_validate | Stored JSON parsed back into a typed draft each turn |

> Phase 1 note: no webhook signature validation here (that is Phase 2). The orchestrator trusts its caller to have authenticated the sender; Phase 1 tests inject sender/message_id directly. This assumption is documented so Phase 2 wires the allowlist + signature check ahead of handle_message.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | duplicate webhook delivery → duplicate write | mitigate | last_message_id idempotency check is step 2, before any state read (Pitfall F); CONV-02 test asserts no advance/no write |
| T-04-02 | Tampering | concurrent messages corrupt conversation state | mitigate | per-sender SELECT ... FOR NO KEY UPDATE via with_for_update(key_share=True) serializes handlers (CONV-03); test spy-asserts the lock arg |
| T-04-03 | Tampering | malformed draft_gasto JSON injected | mitigate | DraftGasto.model_validate wrapped in try/except → reset to idle on ValidationError; all fields Optional so schema evolution does not crash (Pitfall D) |
| T-04-04 | Spoofing | stale state replayed after restart | mitigate | DB-backed last_message_id + updated_at timeout reset (CONV-04) |
| T-04-05 | Tampering | LLM-driven write without confirmation | mitigate | confirm gate is a deterministic AFFIRMATIVE string match; GPT never invoked at confirm (D-05); save_gasto reachable only through this gate |
| T-04-SC | Tampering | pip installs | accept | No new packages (RESEARCH Package Legitimacy Audit: none); no install task |
</threat_model>
