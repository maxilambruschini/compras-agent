<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ExtractionService → OpenAI API | Outbound call carries the API key bound to the AsyncOpenAI client. Service code never references the raw key. |
| OpenAI response → ExtractionService | GPT-4o output is constrained by Structured Outputs; refusal text is untrusted, must not be echoed verbatim to clients beyond raising the typed exception. |
| Image bytes → GPT-4o (via base64) | Image content is untrusted; prompt injection attempts inside the image are mitigated by Structured Outputs schema constraint. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-02 | Information Disclosure | ExtractionService error logging | mitigate | test_log_payload_does_not_contain_secrets asserts none of the conftest secret values ever appear in captured log entries; service code grep gate proves no `openai_api_key` / `whatsapp_*` reference. |
| T-02-05 | Tampering | Prompt injection in invoice image | mitigate | Structured Outputs JSON Schema constrains GPT-4o output to ExtractedInvoice fields. Refusals raise ExtractionRefusalError; the refusal string is exposed only via exception .args, not returned to the HTTP client (FastAPI default 500 response). |
| T-02-06 | Repudiation | Extraction failure with no audit trace | mitigate | structlog logs "extraction.start", "extraction.complete" (with confidence + status + image_path), and "extraction.failed" (with stage + error string) — all bound with filename basename. Satisfies VAL-05. |
| T-02-07 | Spoofing | Caller could pass crafted filename containing newlines / unicode escape sequences | accept | filename is only used in (a) os.path.basename for storage (already sanitized by Plan 01 T-02-01) and (b) structlog log payload (escaped by structlog renderer). No SQL or shell context. |

</threat_model>
