LocalStorageBackend (this plan creates) — BASENAME-ONLY sanitization model:
- __init__(self, root: str) -> None
- save(self, data: bytes, filename: str) -> str
- Contract: The caller (ExtractionService) is responsible for pre-constructing the relative path "{uuid}/{basename}" before calling save(). LocalStorageBackend.save() does NOT preserve directory components from the input — it sanitizes filename with os.path.basename() (stripping any "../" or "/" segments entirely) and writes to "{root}/{basename(filename)}". This intentionally collapses any path-traversal attempt to a single filename in the root directory. The UUID-namespace prefix is the service layer's responsibility, NOT storage's. (T-02-01 mitigation simplified per Phase 2 review.)
- Note: This means save(b"x", "uuid-xxx/photo.jpg") writes to "{root}/photo.jpg" and returns "photo.jpg" — NOT preserving the uuid-xxx/ prefix. The service layer must pass the basename-only filename OR adopt a different storage backend strategy. For Phase 2 MVP skeleton, this is acceptable because the service does not yet read back saved images; image_path is a write-receipt only.
- Wait — refinement: the service still needs to namespace by UUID to avoid collisions. So in practice, save() takes a filename that is already just the basename, and the service constructs the relative output path itself via `os.path.join(invoice_uuid, basename)` and passes ONLY that joined path. To preserve the UUID directory while keeping save() simple, save() will: (1) accept a relative path, (2) split into dirname/basename via os.path.split, (3) sanitize basename via os.path.basename, (4) sanitize dirname by extracting ONLY the first path component if it does NOT start with "." (drops any traversal segments), (5) join + write. This is still basename-only sanitization at each level — no realpath reconstruction.
- Actual implementation contract (FINAL):
  - parts = filename.replace("\\", "/").split("/")
  - safe_parts = [p for p in parts if p and p not in (".", "..")]
  - if not safe_parts: raise ValueError("filename has no valid component after sanitization")
  - relative_path = "/".join(safe_parts)
  - full_path = os.path.join(self._root, relative_path)
  - Defense-in-depth: assert os.path.commonpath([os.path.realpath(full_path), os.path.realpath(self._root)]) == os.path.realpath(self._root); raise ValueError otherwise.
  - os.makedirs(os.path.dirname(full_path), exist_ok=True) when dirname non-empty.
  - Write bytes; return relative_path.

- async extract(self, image_bytes: bytes, filename: str) -> ExtractionResult
-   1. invoice_uuid = str(uuid.uuid4())
-   2. safe_filename = os.path.basename(filename)
-   3. relative_path = self._storage.save(image_bytes, f"{invoice_uuid}/{safe_filename}")

-   7. score = compute_confidence(parsed) ; status = assign_status(score, settings.confidence_threshold)
-   8. return ExtractionResult(invoice=parsed, confidence_score=score, status=status, image_path=relative_path)
