    The complete Admin UI with:
    - Invoice list page (/) with FilterToolbar, InvoiceTable (amber rows for pending_review), Pagination
    - Invoice detail page (/invoices/:id) with side-by-side DataPanel + ImagePanel layout
    - Edit document modal (pre-filled, saves via PATCH)
    - Edit line item modal (pre-filled, saves via PATCH)
    - Confirm/Reject buttons (pending_review only) via PATCH status
    - Delete with inline confirmation strip (removes DB row, retains image file)
    - Full backend: 7 API endpoints, CORS, path traversal protection
    - Responsive design: mobile-first with md: breakpoint adaptations
  </what-built>
  <how-to-verify>
    Start the stack:
    1. `docker compose up` (from project root) — starts Postgres and backend on :8000
    2. `cd frontend && npm run dev` — starts Vite dev server on :5173
    3. Open http://localhost:5173 in a browser

    Verify these flows:

    **UI-01 / UI-06 — Invoice List with filter and highlight:**
    - The page loads with title "Facturas" and a FilterToolbar
    - If there are pending_review rows, they have an amber/yellow background (#fef3c7)
    - StatusBadge shows "Revisar" (amber), "Guardado" (gray), "Confirmado" (green), "Rechazado" (red)
    - Select "Revisar" from the Estado dropdown and click "Filtrar" — only pending_review rows appear
    - Clear filters — all invoices return
    - Pagination "Siguiente →" enabled when more than 20 invoices exist

    **UI-02 — Search:**
    - Type a known proveedor name in the Buscar field and click "Filtrar" — matching invoices appear
    - Type a partial document number — matching invoices appear
    - Type a string that matches nothing — "Sin facturas" empty state shows

    **UI-03 — Detail page with image:**
    - Click a row — navigates to /invoices/:id
    - Left panel shows all document fields (Proveedor, Tipo, Número, Fecha, CUIT, CAE, Estado, Confianza)
    - Right panel shows the invoice image (or "Sin imagen" if no image_path)
    - "← Volver a la lista" link returns to list; browser back button also works
    - At desktop width (≥768px): two-column side-by-side layout; image panel is sticky
    - At mobile width (<768px): single column, DataPanel above ImagePanel

    **UI-04 — Edit document fields:**
    - Click "Editar documento" on the detail page — modal opens with all fields pre-filled
    - Modify the Proveedor field — click "Guardar cambios"
    - Modal closes; detail page shows updated Proveedor value
    - Click "Editar" on a line item row — line item modal opens pre-filled
    - Modify Descripción — click "Guardar cambios"
    - Modal closes; line item table shows updated value

    **UI-05 — Delete:**
    - Click "Eliminar factura" on the detail page
    - Inline confirmation strip appears below ActionBar (NO modal — it's an inline strip)
    - Strip shows "¿Eliminar esta factura?" heading, body text about file retention, "Sí, eliminar" and "Cancelar" buttons
    - Click "Cancelar" — strip disappears
    - Click "Eliminar factura" again — strip reappears
    - Click "Sí, eliminar" — navigates back to list; the deleted invoice is gone

    **UI-06 — Confirm/Reject flow:**
    - Navigate to a pending_review invoice detail page
    - "Confirmar" and "Rechazar" buttons are visible in the ActionBar
    - Navigate to a confirmed or rejected invoice — those buttons are NOT visible
    - Click "Confirmar" on a pending_review invoice — status updates to "Confirmado"; buttons disappear

    **Responsive check:**
    - Resize browser to mobile width (<768px)
    - List page: FilterToolbar stacks vertically; table shows only Proveedor, Estado, and Creado columns
    - Detail page: single column; DataPanel above ImagePanel (not side by side)
