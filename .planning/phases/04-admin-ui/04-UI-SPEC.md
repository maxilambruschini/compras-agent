# Phase 4 — UI Design Contract

> Visual and interaction contract for the Admin UI phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | Tailwind CSS v4 |
| Preset | shadcn/ui default (CSS variables enabled) |
| Component library | shadcn/ui — `Button`, `Dialog`, `Badge`, `Input`, `Label`, `Select`, `Table`, `Alert`, `Separator` |
| Icon library | lucide-react (installed as shadcn peer dep — use sparingly, prefer text labels) |
| Font | system-ui, 'Segoe UI', Roboto, sans-serif (existing `--sans` variable, kept in `index.css`) |

**Installation (planner must include as Wave 1 tasks):**
- `npm install tailwindcss @tailwindcss/vite` + configure `vite.config.ts` with `@tailwindcss/vite` plugin
- `npx shadcn@latest init` — style: default, CSS variables: yes, tailwind config: auto (v4)
- Add individual components: `npx shadcn@latest add button dialog badge input label select table alert separator`
- Do NOT bulk-install all shadcn components — only the list above

**Rationale (source: CONTEXT.md D-15, D-16):** Tailwind CSS v4 provides utility-first styling for the dense admin layout (tables, forms, modals). shadcn/ui provides accessible, unstyled-first primitives for Dialog (edit modals), Button variants (primary/destructive/outline), Badge (status chips), Table, and form controls — eliminating hand-rolling of focus management and ARIA for those components. Custom components (InvoiceTable row behavior, ImagePanel, DeleteConfirmation strip, Pagination) are hand-written on top of these primitives.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, badge inner padding, table cell icon spacing |
| sm | 8px | Input label-to-field gap, button icon-to-text gap, table cell vertical padding |
| md | 16px | Default element spacing, modal body padding, form field gap, table cell horizontal padding |
| lg | 24px | Card/panel inner padding, modal header padding, section separation within detail page |
| xl | 32px | Layout column gap (list page toolbar row), modal footer padding-top |
| 2xl | 48px | Page header height (top nav bar including logo + padding) |
| 3xl | 64px | Not used in this phase |

Exceptions:
- List table row height: 44px minimum (touch target compliance even for desktop — prevents misclick on status highlight)
- Modal max-width: 560px (document edit), 640px (line-item edit — wider for numeric columns) — layout constraints, not spacing tokens; exempt from the 4px grid requirement
- Detail page split: `grid-template-columns: 1fr 1fr` with 32px gap; image panel is `position: sticky; top: 24px`

**Accent (`#2563eb`) reserved for:**
- Primary action buttons: "Confirmar" (Confirm) button on pending_review invoices
- "Guardar" (Save) button inside edit modals
- Active/focused filter pill or selected status chip
- Pagination active page indicator

Accent is NOT used for: table row hover, column headers, status badges, destructive actions, navigation links.

**Destructive (`#dc2626`) reserved for:**
- "Eliminar" (Delete) button on detail page
- Destructive confirmation text inside delete confirmation prompt

**Pending highlight (`#fef3c7`) reserved for:**
- `pending_review` row background in the invoice list table — the entire `<tr>` background (source: CONTEXT.md D-14)

**"Rechazar" (Reject) button styling:**
- Uses a neutral/outline style (not destructive red) — rejecting an invoice is a workflow action, not data deletion. Use `--text-h` border + text with hover to `--destructive`.

## Interaction Contracts

### Edit Flow (both modals)

1. User clicks "Editar documento" or "Editar" on a line item row.
2. Modal opens; all fields pre-filled with current values from TanStack Query cache.
3. User edits one or more fields.
4. User clicks "Guardar" — button shows spinner, is disabled.
5. On success: modal closes, TanStack Query invalidates `["invoice", id]`, page re-renders.
6. On error: modal stays open, ErrorBanner appears inside modal with "No se pudo guardar. Intente nuevamente."

### Confirm / Reject Flow

1. Visible only when `invoice.status === "pending_review"` (source: CONTEXT.md D-07).
2. One-click, no reason required (source: CONTEXT.md D-05).
3. "Confirmar" button is accent blue; "Rechazar" is outline with hover-to-destructive.
4. On click: button shows spinner + disabled. PATCH to `/invoices/{id}/status`.
5. On success: TanStack Query invalidates `["invoice", id]` + `["invoices"]`; ActionBar re-evaluates condition and hides both buttons.
6. On error: ErrorBanner below ActionBar: "No se pudo actualizar el estado. Intente nuevamente."

### Delete Flow

1. User clicks "Eliminar" on detail page.
2. Inline confirmation strip appears immediately below ActionBar (no modal — faster, lower friction for an action the manager consciously navigated to).
3. User clicks "Sí, eliminar" — DELETE to `/invoices/{id}`.
4. On success: navigate to `/` (list); TanStack Query invalidates `["invoices"]`.
5. On error: ErrorBanner: "No se pudo eliminar la factura. Intente nuevamente."

### Navigation

- List page → detail page: click anywhere on a table row (`cursor: pointer`, no explicit button).
- Detail page → list page: "Volver" link/button top-left of DataPanel. Browser back button also works (source: CONTEXT.md D-10 — bookmarkable).

### Filter + Search

- Filters are applied on "Filtrar" button click (not on-change) to avoid excessive API calls.
- Search (`q` param) and status/proveedor/date filters all submit together as a single GET request.
- Clearing all filters and clicking "Filtrar" returns to unfiltered list.

### Pagination

- Offset-based: `?page=1&page_size=20` (source: CONTEXT.md Claude's Discretion — offset simpler for v1).
- "Anterior" disabled on page 1; "Siguiente" disabled when returned items < page_size.
- Changing filters resets to page 1.

## Responsive Design Contract

**Breakpoints (Tailwind defaults):** `sm` = 640px, `md` = 768px, `lg` = 1024px. Mobile-first: base styles target mobile, `md:` prefix adds desktop overrides.

### List Page

| Element | Mobile (< md) | Desktop (≥ md) |
|---------|--------------|----------------|
| FilterToolbar | `flex flex-col gap-2` — inputs stack vertically, "Filtrar" full-width | `flex flex-row flex-wrap gap-2` — single row |
| InvoiceTable columns | Proveedor, Estado, Total only (`hidden md:table-cell` on Tipo, Número, Fecha) | All 6 columns visible |
| Table cell padding | `px-3 py-2` | `px-4 py-3` |

### Detail Page

| Element | Mobile (< md) | Desktop (≥ md) |
|---------|--------------|----------------|
| Page grid | `grid grid-cols-1` — DataPanel then ImagePanel stacked | `grid grid-cols-2 gap-8` |
| ImagePanel position | static (scrolls with page) | `sticky top-6` |
| ActionBar buttons | `flex flex-col w-full gap-2` | `flex flex-row gap-3` |

### Modals

| Element | Mobile (< md) | Desktop (≥ md) |
|---------|--------------|----------------|
| Dialog width | `w-full mx-4` (near full-screen) | `max-w-[560px]` / `max-w-[640px]` |
| Dialog position | bottom-anchored or centered | centered |

### Touch Targets

All interactive elements (buttons, table rows, filter inputs, pagination controls) maintain a minimum height of **44px** on all screen sizes — enforced via `min-h-[44px]` Tailwind class where shadcn defaults are smaller.

## Accessibility Baseline

- All interactive elements reachable by Tab; logical tab order within modals.
- Modal uses `role="dialog"` + `aria-labelledby` pointing to modal title; focus trapped inside while open.
- Status badges use `aria-label` with full status word (not just color).
- Delete confirmation: "Sí, eliminar" button has `aria-describedby` pointing to the confirmation body text.
- Color is not the sole differentiator for `pending_review` — the status badge text ("Revisar") also conveys state.
- Image panel: `<img alt="Factura original de {proveedor} — {numero_documento}">`.
