<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| caller → POST /gastos/prompt | Untrusted HTTP request crosses here; bearer token + Pydantic body are the gate |
| endpoint → Conversation row | Concurrent writer (inbound webhook orchestrator) may race; row lock serializes |
| endpoint → Twilio (WhatsApp) | Outbound send may fail (closed 24h window); must not corrupt committed state |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-A1 | Spoofing / Elevation | verify_token bearer check | mitigate | secrets.compare_digest constant-time compare (Task 1); covered by test_valid/missing/wrong_token |
| T-03-A2 | Elevation of Privilege | empty GASTOS_PROMPT_TOKEN | mitigate | fail-closed 401 before compare (Task 1, Pitfall 5); test_empty_configured_token_denies |
| T-03-A3 | Information Disclosure | token in logs | mitigate | log "auth.invalid_token" with no credential value; token never logged (Task 1) |
| T-03-R1 | Tampering | state race endpoint vs webhook | mitigate | pg_insert ON CONFLICT DO NOTHING + SELECT FOR NO KEY UPDATE before read/set (Task 1); test_row_lock_issued |
| T-03-R2 | Tampering / Denial | send-then-rollback double-prompt | mitigate | _safe_send strictly AFTER commit; send failure logged not raised (Task 1, Pitfall 1/C) |
| T-03-V1 | Tampering | phone_number injection | mitigate | Pydantic PromptRequest(phone_number: str); value passed to provider.send_message as data, no shell/SQL interpolation (parameterized ORM) |
| T-03-SC | Tampering | npm/pip installs | accept | No new packages (RESEARCH §Package Legitimacy Audit: satisfied by absence; secrets/HTTPBearer are stdlib/FastAPI already pinned). No install task. |
</threat_model>
