<objective>
Wire the live `/gastos/webhook` Twilio router — a structural clone of the v1.0 invoice webhook — that validates the signature, gates on the allowlist, returns HTTP 200 before any DB/GPT work, then runs `ConversationOrchestrator.handle_message` in a background task. When a ticket photo is present, the background task downloads it, runs the magic-byte guard, stores it via LocalStorageBackend, runs `TicketVisionService.extract_amount` (D-02), and feeds the stored path + amount into the orchestrator's media entry path (D-06 router side). Mount the router under the `AGENT_MODE=='gastos'` seam in main.py (D-09).

Purpose: end-to-end WhatsApp transport for the gasto flow over Twilio.
Output: `backend/app/routers/gastos.py` (new) + the main.py mount, with webhook tests.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Twilio → /gastos/webhook | untrusted inbound HTTP; signature + allowlist gate before any work |
| remote media URL → backend | provider-hosted media downloaded into the worker |
| stored filename → filesystem | MessageSid-derived path written to local storage |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-W1 | Spoofing | webhook signature | mitigate | `provider.validate_signature(effective_url, form_data, sig)` → 401 before any DB/GPT; effective URL uses WEBHOOK_BASE_URL when set (whatsapp.py:166-183 clone) |
| T-02-W2 | Elevation of Privilege | allowlist bypass | mitigate | SenderAllowlist gate (is_active==True) BEFORE scheduling the orchestrator; non-allowlisted → reply + 200, no orchestrator call, no DB write (success criterion 3) |
| T-02-W3 | Repudiation/Replay | duplicate MessageSid | mitigate | router `_processed_message_sids` fast-path + orchestrator DB `last_message_id` (source of truth) — neither advances state nor writes twice (success criterion 4) |
| T-02-W4 | Tampering | malicious/oversized/spoofed-MIME media | mitigate | two-layer guard: MIME check pre-download + `_validate_image_bytes` magic bytes post-download; bad bytes never reach vision or storage (T-3-09) |
| T-02-W5 | Spoofing/SSRF | media URL points at an internal host | mitigate | only download URLs from a signature-validated Twilio webhook via `provider.download_media`; no user-supplied arbitrary URL is fetched |
| T-02-W6 | Tampering | path traversal in stored filename | mitigate | filename derived from MessageSid; LocalStorageBackend.save is path-traversal-safe (storage.py:55-119, sanitizes basename) |
| T-02-W7 | Denial of Service | Twilio 5s timeout | mitigate | HTTP 200 returned before any DB/GPT work; processing runs in an asyncio background task (success criterion 5) |
| T-02-W8 | Information Disclosure | API key / image leakage in logs | mitigate | never log the OpenAI API key or raw image bytes; structlog binds sender/message_sid only |
| T-02-SC | Tampering | npm/pip installs | mitigate | No new packages introduced (openai/fastapi/sqlalchemy already pinned in CLAUDE.md); slopcheck N/A this plan |
</threat_model>
