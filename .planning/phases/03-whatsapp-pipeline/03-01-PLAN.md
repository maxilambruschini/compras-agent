<objective>
Deliver the thinnest end-to-end WhatsApp inbound slice: a Twilio webhook that validates HMAC-SHA256 signatures, gates on the sender allowlist, sends an immediate Spanish acknowledgement via TwilioProvider, and returns HTTP 200 within the Twilio 5-second window. The provider abstraction (`WhatsAppProvider` Protocol + `TwilioProvider` + `MetaCloudProvider` stub) is the architectural spine that Plan 02 will extend with the background extraction task.

Purpose: Make WA-01 (acknowledgement), WA-02 (allowlist rejection), INF-02 (HMAC), and INF-04 (response < 5s) work end-to-end with a real Twilio sandbox before any extraction wiring exists. After this plan, an allowlisted employee can send a photo and receive `✅ Factura recibida. Procesando...`; a non-allowlisted sender receives `❌ Este número no está autorizado para enviar facturas.` The background task hook fires `asyncio.create_task` with a placeholder coroutine that just logs and returns — Plan 02 replaces this placeholder with the real `process_invoice` pipeline.

Output: New provider package, Twilio webhook router, Alembic UNIQUE constraint migration, Wave 0 test scaffolds, requirements.txt update, docker-compose env var additions.
</objective>
