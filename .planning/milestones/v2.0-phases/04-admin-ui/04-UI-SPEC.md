# Phase 4 — UI Design Contract

> Visual and interaction contract for the read-only Admin UI (Gastos + Cierres).
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (vanilla CSS custom properties) |
| Preset | not applicable |
| Component library | none — hand-rolled CSS using existing `index.css` token system |
| Icon library | none — use Unicode/emoji indicators for ticket presence (e.g. "📎" or "✓") or a text badge |
| Font | system-ui, 'Segoe UI', Roboto, sans-serif (inherited from existing `--sans` token) |

**Source:** Existing `frontend/src/index.css` already defines a full light/dark token set.
All new components MUST consume the CSS custom properties defined there — do not introduce
any Tailwind, shadcn, or third-party CSS framework. Extend `index.css` with new component
rules; do not create a conflicting parallel token system.

---

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, table cell micro-spacing |
| sm | 8px | Inline element gaps, compact row padding |
| md | 16px | Default cell padding, form control gaps, card inner padding |
| lg | 24px | Section padding, nav height inner padding |
| xl | 32px | Page-level horizontal padding, between major sections |
| 2xl | 48px | Top-of-page breathing room |
| 3xl | 64px | Not used in this phase |

Exceptions: Touch targets on filter controls minimum 44px tall (accessibility).

## Layout

**Navigation:** Two-tab top navigation bar. Tabs: "Gastos" | "Cierres de Caja".
- Nav bar height: 48px.
- Active tab indicated by bottom border 2px solid `var(--accent)` + `var(--text-h)` color.
- Inactive tab: `var(--text)` color, no border.
- Nav background: `var(--bg)`, border-bottom: 1px solid `var(--border)`.
- No sidebar. No hamburger menu. No drawer. Tabs are always visible.

**Page width:** Max-width 1126px centered, inheriting the existing `#root` constraint.

**Three routes:**
| Path | Component | Title |
|------|-----------|-------|
| `/` or `/gastos` | GastosListPage | Gastos |
| `/gastos/:id` | GastoDetailPage | Detalle de Gasto |
| `/cierres` | CierresListPage | Cierres de Caja |

**Source:** `04-CONTEXT.md` — three views locked. Layout shape (top tabs) chosen for
simplicity: two destinations, always visible, zero discoverability cost for internal tool.

| Body | 16px | 400 | 1.5 | Default text, table cell content, filter labels |
| Label | 14px | 400 | 1.4 | Secondary metadata (sender_phone, table header labels) |
| Heading | 20px | 500 | 1.2 | Page titles ("Gastos", "Cierres de Caja") |

## Component Contracts

### Table (shared pattern — used by both list pages)

```
- Full-width table (100% of content area)
- thead: background var(--secondary-surface), border-bottom 1px var(--border)
- thead th: 14px / weight 400 / var(--text) / uppercase / letter-spacing 0.05em / padding 8px 16px
- tbody tr: border-bottom 1px var(--border) / background var(--bg)
- tbody tr:hover: background var(--secondary-surface) — no other hover effect
- tbody td: 16px / weight 400 / var(--text-h) / padding 12px 16px
- On GastosListPage: entire row is clickable → navigates to /gastos/:id
  Cursor: pointer on the row
- On CierresListPage: rows are NOT clickable. No pointer cursor. No hover action.
- Newest-first row order (created_at desc — enforced by backend)
```

### Filter Bar (GastosListPage only)

```
- Horizontal strip above the table, background var(--secondary-surface),
  border-bottom 1px var(--border), padding 12px 16px, gap 16px
- Controls: [Desde: date input] [Hasta: date input] [Buscar: text input]
  [Limpiar button]
- Date inputs: type="date", 44px min-height, border 1px var(--border),
  border-radius 6px, padding 8px 12px, font 16px var(--sans)
- Search input: type="text", placeholder "Buscar concepto...",
  min-width 220px, same height/border/padding as date inputs
- Limpiar button: text-only, 16px, var(--text), no background, underline on hover —
  resets all filter state to empty
- Focus: outline 2px solid var(--accent), outline-offset 2px (all inputs)
- Filter is client-side if results ≤ 200 rows; query-param driven to backend
  if results page grows. Planner decides implementation; visual contract unchanged.
```

### Monto Display

```
- Format: Argentine locale — thousands separator ".", decimal separator ","
  Prefix: "$" (no space before number)
  Example: $1.234,56 — always show 2 decimal places
- Font: inherit body (16px, weight 400)
- No special color. No bold. Right-aligned in table column.
- Decimal serialization from backend: string preferred (preserves precision);
  UI parses with parseFloat then formats — never use toLocaleString('es-AR')
  without a fallback (locale support is inconsistent); use manual formatting instead.
```

### Ticket Indicator (GastosListPage — "Ticket" column)

```
- Has ticket: badge pill, text "con ticket", background var(--accent-bg),
  border 1px var(--accent-border), color var(--accent), font 12px, border-radius 9999px,
  padding 2px 8px
- No ticket: plain text "—", color var(--text), no badge
- Column header: "Ticket"
- Column width: min-content (shrink-to-fit)
```

### Ticket Image (GastoDetailPage)

```
- If ticket_image_path present: display inline image below the field list.
  max-width: 100%, max-height: 480px, object-fit: contain
  Border: 1px solid var(--border), border-radius: 8px
  Click → opens image in a new browser tab (window.open URL, target="_blank")
  No modal overlay (simpler for demo; reduces implementation scope)
- If no ticket: show label "Sin ticket adjunto" in var(--text), 14px
- Image is fetched from GET /api/gastos/:id/ticket via the API client.
  Render as <img src={ticketUrl}> where ticketUrl is the streaming endpoint URL.
```

### Loading State

```
- Inline spinner: a single CSS keyframe-animated ring, 24px diameter,
  border 3px solid var(--border), border-top-color var(--accent),
  animation: spin 0.8s linear infinite
- Shown centered in the content area (full-page replacement while list loads)
- No skeleton screens (keeps implementation minimal for demo)
```

### Empty State

```
- Centered vertically in content area (min-height 240px)
- Icon: none (text only)
- Heading: see Copywriting Contract
- Body: see Copywriting Contract
- No illustration, no CTA button (read-only UI, nothing to create)
```

### Error State

```
- Inline banner below the filter bar / above the table
- Background: rgba(239,68,68,0.08), border-left 3px solid #dc2626
- Padding: 12px 16px, border-radius 4px
- See Copywriting Contract for copy
- No retry button — user refreshes manually (demo simplicity)
```

### Detail Page Layout (GastoDetailPage)

```
- Back link: "← Volver a gastos" top-left, var(--accent) color, 14px,
  navigates to /gastos (preserves filter state via react-query cache)
- Field list: two-column definition list (dl > dt / dd pairs)
  dt: 14px / var(--text) / weight 400 / text-align right / width 140px / padding-right 16px
  dd: 16px / var(--text-h) / weight 400
  Gap between rows: 12px
- Fields displayed (in order):
  1. Fecha
  2. Concepto
  3. Monto
  4. Ticket
  5. Registrado por (sender_phone)
  6. Creado (created_at — formatted as "31 may 2026, 14:23")
- Ticket image section below the field list, separated by 24px gap + 1px border-top
```

## Interaction States

| State | Trigger | Visual |
|-------|---------|--------|
| Nav tab — active | Current route matches tab path | Bottom border 2px var(--accent), text var(--text-h) |
| Nav tab — inactive | Other route | No border, text var(--text) |
| Nav tab — hover | Mouse over | Text var(--text-h), cursor pointer |
| Table row — hover (Gastos) | Mouse over clickable row | Background var(--secondary-surface), cursor pointer |
| Table row — hover (Cierres) | Mouse over | Background var(--secondary-surface), cursor default |
| Filter input — focus | Tab/click | outline 2px solid var(--accent), outline-offset 2px |
| Filter input — filled | Has value | No special style beyond default |
| Back link — hover | Mouse over | underline |
| Ticket image — hover | Mouse over | cursor pointer (indicates clickable) |
| Page — loading | Data fetch in flight | Spinner centered in content area, table/content hidden |
| Page — error | Fetch rejected | Error banner shown, empty content area |
| Page — empty | Fetch resolved, 0 rows | Empty state shown |
| Page — populated | Fetch resolved, ≥1 rows | Table shown |

## Responsive Breakpoints

This is an internal desktop tool (managers/accountants at a register or office computer).
Mobile is not a primary target for v2.0 demo.

| Breakpoint | Behavior |
|------------|----------|
| ≥ 768px | Full table layout as specified above |
| < 768px | Table columns reduce: hide "Registrado por" column. Filter bar wraps to two rows. All other layout unchanged. |

No card-flip at mobile. No drawer navigation. Horizontal scroll allowed on small screens
if table overflows (overflow-x: auto on table wrapper).

## Implementation Notes for Executor

1. **CSS extension strategy:** Add a new `src/admin.css` (or component-scoped CSS modules
   if executor prefers) — do NOT rewrite `index.css`. Import the new stylesheet in `main.tsx`.
2. **Secondary surface token:** Add `--secondary-surface: #f7f6f9` (light) and
   `--secondary-surface: #1e1f28` (dark) to `:root` in `index.css` light/dark blocks
   before using it elsewhere.
3. **Money formatter:** Implement a `formatARS(value: string | number): string` utility
   that produces `$1.234,56` format. Use manual string manipulation — do not rely on
   `Intl.NumberFormat` with `es-AR` locale (inconsistent mobile browser support).
4. **Date formatter:** Implement `formatDate(iso: string): string` that produces
   `"31 may 2026"` (day month-abbrev year, Spanish). Month abbreviations:
   ene feb mar abr may jun jul ago sep oct nov dic.
5. **Routing:** React Router 7 `createBrowserRouter` + `RouterProvider`. Route `/`
   redirects to `/gastos`.
6. **react-query keys:** `["gastos"]`, `["gastos", id]`, `["cierres"]`.
   `staleTime: 30_000` (30 seconds) — sufficient freshness for a demo tool.
7. **Ticket image URL:** Constructed as `${VITE_API_URL}/gastos/${id}/ticket` and used
   directly as the `<img src>` attribute. The browser's cookie/session context is open
   (no auth), so the streaming endpoint responds directly.
8. **CORS:** FastAPI must allow `http://localhost:5173` in dev. Executor adds
   `CORSMiddleware` in `create_app()`.
